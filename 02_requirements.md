## 1 - Обязательные требования (MUST)

| ID   | Требование                                                                                                              | Критерий приёмки (Как проверить)                                                                                                                           |
| ---- | ----------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 01   | **Импорт файлов**<br>Пользователь должен иметь возможность перетащить (D&D) изображение или папку в окно приложения.    | Файл перетащен -> в базе появилась запись -> карточка отображается в сетке. Файл на диске остался на месте.                                                |
| 02   | **Генерация превью**<br>При импорте приложение должно создавать сжатую копию (thumbnail) изображения в служебной папке. | Исходник весит 10Мб -> в папке приложения появился файл превью весом <200Кб. Сетка не лагает при скролле.                                                  |
| 03   | **Извлечение цвета**<br>При импорте приложение должно определять доминантные цвета изображения.                         | В свойствах карточки после импорта виден набор HEX-кодов (палитра).                                                                                        |
| 04   | **Фильтрация по цвету**<br>Пользователь может выбрать цвет в фильтре, и сетка покажет подходящие изображения.           | Выбран желтый цвет -> в сетке остались только картинки с желтыми оттенками.                                                                                |
| 05   | **Быстрый просмотр**<br>Пользователь может открыть изображение на весь экран нажатием клавиши Space.                    | Выделена карточка -> Нажат Space -> Открылось модальное окно с оригиналом изображения.                                                                     |
| 06.1 | **Автодобавление тегов**<br>Начальный тег должен появлятся сразу с импортом файла, исходя из расширения файла.          | Файл импортирован в программу -> Тег присвоился исходя из расширения файла -> ег сохранился после перезапуска приложения -> Поиск по тегу находит элемент. |
| 06.2 | **Добавление тегов**<br>Пользователь может присвоить произвольные текстовые теги изображению.                           | Тег введен в инпут -> Тег сохранился после перезапуска приложения -> Поиск по тегу находит элемент.                                                        |
| 07   | **Обработка битых ссылок**<br>Если исходный файл удален, приложение должно визуально это показать.                      | Исходный файл удален через Проводник -> В приложении на карточке появилась иконка "Broken Link".                                                           |

## 2 - Желательные требования (NICE)

1. Создание виртуальных папок внутри приложения (например, "Логотипы", "UI Киты").
2. Интерфейс по умолчанию темный (Dark Mode), но желательна возможность переключения.
3.  Возможность перетащить картинку ИЗ приложения в Photoshop или Figma.
4.  Сортировка по дате добавления, по имени, по размеру.
