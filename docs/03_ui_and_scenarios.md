## 1 - Список экранов / Окон

### 1.1 - Main Dashboard (Главное окно)

Единое рабочее пространство без лишних панелей.

* **Top Bar (Верхняя панель):**
    * Window Controls: Кастомные кнопки управления окном (Close, Minimize, Maximize) в стиле macOS/Modern Windows.
    * Global Search: Поле ввода "Type to ponder..." с иконкой поиска.
    * Suggested Tags: Горизонтальный ряд быстрых фильтров (например: PNG, SVG, Text, Python).
    * Action Group: Кнопки "Sort" (Сортировка) и "Filter" (Расширенные фильтры).
    * Branding: Логотип Splatera (желтый абстрактный глиф, напоминающий закреплялку).
* **Content Area (Рабочая область):**
    * Masonry Grid (Кирпичная раскладка). Адаптивная сетка, смешивающая карточки изображений и текстовые блоки разной высоты.

### 1.2 - Modals (Всплывающие окна)

1. Filter Panel: Выпадающее меню при нажатии на кнопку "Filter". Содержит:
    *   Color Picker. Фильтрация по цвету
    *   File Types: Чекбоксы (Images, Code, Text).
    *   Date Range: Выбор периода.
2. Quick Look (Быстрый просмотр):
    * Модальное окно на весь экран (Overlay backdrop blur).
    * Для картинок: Изображение вписано в экран.
    * Для кода/текста: Окно с нумерацией строк и подсветкой синтаксиса (Syntax Highlighting), с возможностью скролла.

## 2 - Элементы интерфейса

### 2.1 - Карточка объекта (Grid Item)

Основной атом интерфейса. Единый стиль для всех типов контента.
* **Контейнер:** Скругленные углы (~33px), тонкая обводка.
* **Типы контента:**
    * Image: Превью изображения (`object-fit: cover`).
    * Code/Text: Фрагмент первых 20 строк файла с подсветкой синтаксиса (моноширинный шрифт).
* **Badge (Бейдж):** Лейбл в правом верхнем углу (PNG, JPG, PY, JS, MD).
* **Hover State (Наведение):**
    *  Появление контекстного оверлея.
    *  Отображение названия файла и даты сохранения.
    *  Появление кнопок действий: "Copy to Clipboard", "Open Folder".

### 2.2 - Цветовое кодирование

*  **Background:** Pitch Dark Grey (`#222222`) для приложения и Grey для кнопок и лейблов (`525252`)
*  **Accent:** Splatera Yellow (просто yellow) (`#FFD16D`) - используется для логотипа, активных состояний, фокуса ввода и выделения (selection).
*  **Text:** White (`#FFFFFF`) для заголовков, Dark Grey (`#616161`) для метаданных, Light Gray для текста кнопок и лейблов (`#AAAAAA`)
* **Border** - Акцентного цвета или Light Grey (`#7C7C7C`)

## 3 - Состояния UI

| Состояние                      | Визуальное отображение                                                               | Поведение                                                         |
| :----------------------------- | :----------------------------------------------------------------------------------- | :---------------------------------------------------------------- |
| **Idle (Покой)**               | Статичная сетка. Поле поиска полупрозрачное.                                         | Готовность к вводу или драг-н-дропу.                              |
| **Drag Over (Перетаскивание)** | Вся область сетки затемняется, появляется пунктирная желтая рамка по периметру окна. | Текст по центру: "Drop to stash".                                 |
| **Processing (Обработка)**     | На новых карточках крутится локальный спиннер.                                       | Генерация превью и анализ цвета в фоне.                           |
| **Filter Active (Фильтр)**     | Обводка кнопки "Filter" подсвечена желтым. В сетке только релевантные карточки.      | Скрытые карточки удаляются из DOM или скрываются (display: none). |
| **Selection (Выбор)**          | Карточка получает жирную желтую обводку.                                             | Активирует возможность Drag-out (вытащить файл из приложения).    |

## 4 - Пользовательские сценарии

### Сценарий 1: Импорт смешанного контента (Code + Design)

1. Пользователь выделяет в системе папку проекта, где лежат `.svg` иконки и `.css` стили.
2. Перетаскивает папку в окно Splatera.
3. **Система:**
    * Сканирует файлы.
    * Для `.svg`: рендерит картинку, определяет цвета. Присваивает тег `SVG`.
    * Для `.css`: читает начало файла, подсвечивает синтаксис. Присваивает тег `Code`, `CSS`.
4. Карточки появляются в общей сетке вперемешку, выстраиваясь оптимально по высоте.

### Сценарий 2: Поиск сниппета и копирование

1. Пользователь помнит, что сохранял код сортировки на Python.
2. Вводит в поиск: "sort".
3. Сетка фильтруется. Пользователь видит карточки, в названии или содержании которых есть слово "sort"
4. Наводит курсор -> нажимает кнопку "Copy" (иконка двух квадратов) на оверлее карточки.
5. Код скопирован в буфер обмена.

### Сценарий 3: Визуальный поиск по цвету

1. Пользователю нужен референс в оранжевых тонах.
2. Нажимает "Filter" -> выбирает Оранжевый цвет в color-picker.
3. Сетка показывает только изображения с оранжевой гаммой. Текстовые файлы скрываются (так как у них нет цвета).
4. Пользователь зажимает карточку мышкой и перетаскивает её (Drag-out) прямо в окно Photoshop/Figma.

## 5. Типовые ошибки и сообщения

1. **Слишком большой текстовый файл:**
    * *Условие:* Пользователь кидает лог-файл на 500 Мб.
    * *UI:* Карточка с иконкой файла и надписью "Preview unavailable (File too large)".
2. **Бинарный файл без расширения:**
    * *Условие:* Файл не распознан как изображение или текст.
    * *UI:* Дженерик-карточка с вопросительным знаком. Тег `Unknown`.
3. **Файл удален из системы:**
    * *Условие:* Попытка открыть Quick Look для удаленного файла.
    * *UI:* Карточка становится полупрозрачной (Disabled), поверх иконка "Broken Link". Текст: "File missing".
4. **Пустой поиск:**
    * *Условие:* Фильтр не нашел совпадений.
    * *UI:* Пустой экран с иллюстрацией и текстом "No artifacts found".
